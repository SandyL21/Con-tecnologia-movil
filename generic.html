Etiquetas

Ejemplos de etiquetas

HTML

Etiqueta BODY

Indica la parte del cuerpo del contenido de un documento HTML. Es una etiqueta esencial para cualquier documento ya que indica donde empieza el contenido visible del documento.

Etiqueta HEAD

La parte superior del documento HTML, es donde podremos indicar los metadatos: título del documento, hojas de estilos, javaScript, CSS…

Etiqueta DIV

Un elemento que es usado mayoritariamente para agrupar otros elementos y actuar como plantilla de otros controles.

Etiqueta A

Es una etiqueta que nos ayuda a poder crear un enlace a una página web. El atributo principal de la etiqueta HTML es href, donde pondremos el enlace al que queremos conectar. Otro atributo muy usado es target, el cual nos sirve para indicar si el enlace se abrirá en una nueva ventana o en la misma.

Etiqueta STRONG

Si tienes mucho texto, es importante poder dar énfasis a una parte en concreto, con la etiqueta strong lo podemos hacer.

CSS

Etiqueta width

Indica el ancho del elemento, regularmente es utilizado para images, pero tambien puede ser utilizado para dimensionar diversos elementos de la página.

Etiqueta background-color

Está etiqueta indica el color de fondo que contendrá nuestra página, y puede contener otros atributos así como la alineación de los elementos dentro de la misma.

Etiqueta Color

Está etiqueta es utilizada para indicar el color de cierto elemento, generalmente utilizada para texto.

Etiqueta display

Está etiqueta ayuda a seleccionar el modo de visualización para los elementos.

Etiqueta text-shadow

Esté elementos nos ayuda a colocar una sombra difuminada en el texto dentro del elemento.

HTTP

Etiqueta extensible

Presentadas en la versión HTTP/1.0, las cabeceras de HTTP, han hecho que este protocolo sea fácil de ampliar y de experimentar con él. Funcionalidades nuevas pueden desarrollarse, sin más que un cliente y su servidor, comprendan la misma semántica sobre las cabeceras de HTTP.

Etiqueta sencillo

Incluso con el incremento de complejidad, que se produjo en el desarrollo de la versión del protocolo HTTP/2, en la que se encapsularon los mensajes, HTTP esta pensado y desarrollado para ser leído y fácilmente interpretado por las personas, haciendo de esta manera más facil la depuración de errores, y reduciendo la curva de aprendizaje para las personan que empieza a trabajar con él.

Etiqueta Conexiones

Una conexión se gestiona al nivel de la capa de trasporte, y por tanto queda fuera del alcance del protocolo HTTP.

Etiqueta Estados

HTTP es un protocolo sin estado, es decir: no guarda ningún dato entre dos peticiones en la mísma sesión.

Etiqueta Cache

El como se almacenan los documentos en la caché, puede ser especificado por HTTP. El servidor puede indicar a los proxies y clientes, que quiere almacenar y durante cuanto tiempo. Aunque el cliente, también puede indicar a los proxies de caché intermedios que ignoren el documento almacenado.
